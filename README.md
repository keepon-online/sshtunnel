# SSH Tunnel Manager

面向 Linux 和 Windows 的桌面 SSH 本地端口转发管理器，基于 Tauri 2、Rust 和无框架静态前端实现。应用同时提供主窗口工作区和系统托盘入口，适合管理少量但需要长期驻留的 SSH 隧道。

## 当前功能

- 管理多个本地转发配置，字段包括 SSH 主机、端口、用户名、本地监听地址/端口、远端目标主机/端口。
- 支持两种认证方式：
  - 密钥认证：要求提供私钥路径，并按 `ssh -L` 方式直接启动系统 `ssh`。
  - 密码认证：通过 PTY 驱动交互式 `ssh`，密码保存在系统凭据库，不落盘到配置文件。
- 密码认证会附加 `StrictHostKeyChecking=accept-new`，首次连接新主机时自动接受新的 host key。
- 连接状态基于 SSH 输出和退出信号判断，不再把“进程存在”误判为“已连接”。
- 支持 `auto_connect`：应用启动后自动连接被标记的隧道。
- 支持 `auto_reconnect`：隧道异常退出后自动重连；用户主动断开时不会触发重连。
- 主窗口提供 command-center 风格工作区：
  - 左侧隧道列表和新建入口
  - 右侧当前隧道状态、转发信息、认证信息、连接健康摘要
  - 时间线式诊断面板，分开展示状态事件和 SSH 输出
- 支持抽屉式编辑器，保存失败会在抽屉内直接显示错误。
- 支持从桌面端调用系统文件选择器填写私钥路径。
- 支持亮色 / 暗色主题切换，并记住本地主题偏好。
- 支持系统托盘常驻，并在托盘中显示最近 3 条隧道的快捷连接/断开操作。
- 支持查看系统 SSH 是否可用、查看配置文件路径，以及切换应用开机自启。
- Linux CI 会执行校验和 `.deb` 打包；Windows CI 会构建 NSIS 安装包。

## 界面与操作

主窗口目前围绕“选中一个隧道后立即观察和操作”设计：

- 隧道列表：展示所有隧道，选中后刷新右侧工作区。
- 工作区 Hero：显示连接状态，并提供连接、断开、编辑、删除操作。
- 属性区：展示当前转发目标、认证方式以及自动重连/错误摘要。
- 连接时间线：把状态事件与 SSH 输出拆分显示，便于排障。
- 编辑抽屉：用于创建或修改隧道配置，可切换认证方式、填写密码、选择私钥、设置自动连接和自动重连。
- 全局设置弹窗：查看 SSH 可用性、配置路径和开机自启状态。
- 托盘菜单：适合常驻运行时直接连接、断开最近使用的隧道。

## 使用要求

- 系统中必须有可执行的 `ssh` 命令。
- 目标平台当前为 Linux 和 Windows。
- 密码认证依赖系统凭据库：
  - Linux 使用 Secret Service
  - Windows 使用 Credential Manager

## 配置与安全

- 隧道配置写入系统配置目录下的 `sshtunnel-manager/config.json`。
- 密码不会写入配置文件，只会保存到系统凭据库。
- 密码认证配置在保存时必须满足以下条件之一：
  - 本次提交了新密码
  - 系统凭据库中已存在该隧道的密码
- 前端隧道列表按文本节点渲染，避免配置字段被当作 HTML 注入到 WebView DOM。
- 密码认证链路会采集 SSH `stderr` 并写入时间线面板，便于定位认证失败、连接断开和重连行为。

## 开发与验证

如果当前环境中 `cargo` 不在默认 `PATH`，先执行：

```bash
export PATH="$HOME/.cargo/bin:$PATH"
```

常用校验命令：

```bash
node --check web/app.js
cargo test --workspace
cargo check -p sshtunnel-app
```

项目结构：

- `crates/core`：隧道模型、参数校验、SSH 参数与启动计划构造
- `src-tauri`：Tauri 桌面壳、运行时管理、托盘和后端命令
- `web`：无框架静态前端
- `docs/superpowers`：设计文档与实现计划

## CI 与打包

GitHub Actions 当前包含三类任务：

- `ubuntu-check`：前端语法检查、核心测试、桌面壳编译检查
- `ubuntu-deb`：构建 Linux `.deb` 安装包
- `windows-installer`：构建 Windows NSIS `.exe` 安装包

## 当前限制

- 当前定位是本地端口转发管理，不包含远程端口转发或 SOCKS 代理配置。
- Debug/dev 模式下若系统仍弹出控制台窗口，属于开发阶段行为；Windows 发布版已对启动黑框做处理。
- README 仅描述仓库里已经实现的能力，不代表未来规划。
